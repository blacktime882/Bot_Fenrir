# ❌ Ошибка: "not a git repository"

## 🔍 Что это означает?

Когда вы используете команду `/обновить` в боте, он пытается выполнить:
```bash
git pull origin main
```

Но на вашем сервере выскочила ошибка:
```
fatal: not a git repository (or any parent up to mount point /home)
```

**Это значит:** Папка `/home/container` (где находится код) - это **не git репозиторий**.

## 🤔 Почему это произошло?

Есть несколько причин:

### 1. **Код залит просто как файлы** (САМАЯ ЧАСТАЯ)
- Вы использовали File Manager на хостинге
- Вы распаковали ZIP и загрузили файлы
- Вы скопировали файлы через SFTP
- Git не был инициализирован

### 2. **Код был скопирован**
- Вы скопировали файлы с другого места
- История git потеряна

### 3. **Неправильный путь**
- Бот запущен не из корневой папки репозитория

## ✅ Как исправить?

### Вариант 1: Инициализировать Git на сервере (ЛУЧШИЙ)

Если у вас есть консольный доступ (Pterodactyl Panel):

```bash
cd /home/container

# Инициализировать новый git репозиторий
git init

# Добавить удаленный репозиторий
git remote add origin https://github.com/blacktime882/Bot_Fenrir.git

# Загрузить все данные
git fetch origin main

# Переключиться на главную ветку
git checkout main

# Установить зависимости (если не установлены)
npm install
```

**После этого:**
- Команда `/обновить` будет работать
- Бот сможет загружать обновления автоматически
- Вы сможете видеть какие файлы изменились

### Вариант 2: Переиспользовать исходные файлы (Если вариант 1 не работает)

Если консоль не доступна:

1. Удалите все файлы бота (кроме `.env`)
2. Скачайте свежую версию:
   ```
   https://github.com/blacktime882/Bot_Fenrir/archive/refs/heads/main.zip
   ```
3. Распакуйте локально
4. Загрузите файлы на сервер через File Manager
5. Перезагрузите бот

**Минус:** Команда `/обновить` не будет работать

### Вариант 3: Использовать GitHub Actions (Автоматические обновления)

Если хостинг поддерживает вебхуки:

1. Настроить вебхук из GitHub
2. При каждом push он будет автоматически обновлять код
3. Подробнее смотрите [DEPLOYMENT.md](DEPLOYMENT.md)

## 📝 Проверка

Чтобы проверить что всё сработало, в консоли выполните:

```bash
# Проверить что это git репозиторий
ls -la .git

# Проверить ветку
git branch -a

# Проверить последний коммит
git log --oneline -1
```

Должно вывести:
```
On branch main
...886592e (HEAD -> main, origin/main) ...
```

## 🎯 Что дальше?

Теперь команда `/обновить` в боте будет работать:

```
/обновить
```

Бот ответит:
- ✅ "Обновление выполнено!" - если есть новые изменения
- ✅ "Код уже актуален" - если уже на последней версии
- ❌ Сообщение об ошибке - если что-то не так

## 🆘 Если ничего не работает

1. **Проверьте консоль:** Там должны быть логи вроде:
   ```
   [Update] Already up to date
   [Update Success] ...
   ```

2. **Проверьте права доступа:**
   ```bash
   # Проверить владельца файлов
   ls -la /home/container
   
   # Если нужно изменить владельца
   chown -R container:container /home/container
   ```

3. **Проверьте интернет:**
   ```bash
   ping github.com
   ```

4. **Посмотрите полный лог:**
   ```bash
   git pull origin main
   ```
