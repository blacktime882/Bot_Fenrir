# 🚀 Развертывание Bot_Fenrir на хостинге

## Проблема: "not a git repository"

Если вы получите ошибку:
```
fatal: not a git repository (or any parent up to mount point /home)
```

Это означает что код был залит на сервер просто как файлы, а не через `git clone`.

## ✅ Решение

### Вариант 1: Инициализировать Git на сервере (РЕКОМЕНДУЕТСЯ)

Если у вас есть доступ к консоли сервера:

```bash
cd /home/container  # или путь к вашей папке

# Инициализировать git репозиторий
git init

# Добавить удаленный репозиторий
git remote add origin https://github.com/blacktime882/Bot_Fenrir.git

# Загрузить данные
git fetch origin main

# Переключиться на main ветку
git checkout main

# Установить зависимости
npm install
```

Теперь команда `/обновить` будет работать и сможет загружать новые версии кода.

### Вариант 2: Для Pterodactyl Panel

Если вы используете **Pterodactyl Panel**:

1. **Скачайте ZIP файл** с GitHub:
   - Перейдите на https://github.com/blacktime882/Bot_Fenrir
   - Нажмите `Code → Download ZIP`

2. **Распакуйте и залейте** через File Manager:
   - Распакуйте ZIP локально
   - Откройте File Manager в Pterodactyl
   - Удалите старые файлы (если есть)
   - Загрузите новые файлы из распакованной папки

3. **Или используйте git** в консоли Pterodactyl:
   ```bash
   cd /home/container
   git init
   git remote add origin https://github.com/blacktime882/Bot_Fenrir.git
   git fetch origin main
   git checkout main
   npm install
   ```

### Вариант 3: Если у вас есть доступ только через панель

Если вы не можете запустить git команды через консоль:

1. Скачайте последнюю версию с GitHub
2. Распакуйте локально
3. Загрузите файлы на сервер через SFTP или File Manager
4. Перезагрузите бот

## 📝 Конфигурация

После загрузки кода убедитесь что созданы файлы:

```
/home/container/
  ├── .env              (⚠️ ВАЖНО: создайте на основе .env.example)
  ├── .env.example      (шаблон переменных)
  ├── index.js          (главный файл)
  ├── package.json      (зависимости)
  └── ...другие файлы...
```

### Переменные окружения (.env)

Создайте файл `.env` на основе `.env.example`:

```env
DISCORD_TOKEN=your_bot_token_here
DISCORD_GUILD_ID=optional_guild_id
NOTIFY_CHANNEL_ID=your_notification_channel_id
PORT=3000
ENABLE_WEB_SERVER=false
```

## 🔄 Обновление кода

### Если инициализирован Git:

Используйте команду `/обновить` в Discord (только администраторы)

```
/обновить
```

Бот автоматически:
- Загрузит последние изменения
- Перезагрузится (если настроено)
- Покажет результат

### Если Git не инициализирован:

Вручную:
1. Скачайте новую версию с GitHub
2. Загрузите файлы на сервер
3. Перезагрузите бот

## 🧪 Проверка

После развертывания проверьте что бот работает:

```bash
# Проверка конфигурации
cat .env

# Проверка зависимостей
npm list

# Запуск бота (если нужно вручную)
npm start

# Проверка источников данных
node lib/verify-sources.js
```

## 🆘 Часто задаваемые вопросы

**Q: Как понять что бот работает?**  
A: Проверьте что бот:
- Отвечает на команду `/пинг`
- Показывает арбитражи в `/арбитраж`
- Логирует данные в консоль

**Q: Как перезагрузить бота?**  
A: В зависимости от хостинга (Pterodactyl - кнопка "Restart")

**Q: Бот не видит команды**  
A: Команды регистрируются автоматически при старте, подождите 30 секунд

**Q: Ошибка "DISCORD_TOKEN not found"**  
A: Создайте файл `.env` и добавьте токен
